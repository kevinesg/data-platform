# data_warehouse

This dbt project owns warehouse transformation code for the data platform.

The project starts with the generated dbt skeleton only. Sources, models, seeds,
tests, macros, and environment-specific profile configuration are added in the
commits that need them.

Use `../README.md` for dbt component setup, local runtime commands, external
profile setup, and cloud access verification. This nested README documents only
the dbt project directory itself.

Do not place local `profiles.yml`, credentials, compiled artifacts, or dbt logs
in this directory.
